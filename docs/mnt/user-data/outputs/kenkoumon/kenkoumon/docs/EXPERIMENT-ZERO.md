# Experiment Zero — Validation Protocol

## Objective

Validate the core hypothesis with zero infrastructure investment: **Can AI turn a recorded Japanese medical consultation into a structured Japanese-language doctor briefing that physicians find genuinely useful?**

If this experiment fails, no amount of engineering will save the product. If it succeeds, everything else is scaling.

---

## Hypothesis

A Japanese-language doctor briefing generated by AI from a recorded consultation will be:

1. **Accurate** — correctly captures medications, instructions, diagnoses, and key discussion points
2. **Useful** — the treating physician considers it helpful for preparing for the next visit
3. **Worth paying for** — the patient (an expat navigating Japanese healthcare) would pay ¥1,500/month for this service

---

## Protocol

### Step 1: Record

- **What:** Record your next scheduled doctor's appointment
- **How:** Any recording app on your phone (Voice Memos, etc.)
- **Consent:** Verbally ask the doctor: "先生、今日の診察を録音してもよろしいですか？自分の記録のためです。" ("Doctor, may I record today's consultation? It's for my own records.")
- **Output:** An audio file (m4a, mp3, wav — format doesn't matter)
- **Time:** 0 minutes prep

### Step 2: Transcribe

- **What:** Convert Japanese audio to Japanese text
- **How (Option A — API):** OpenAI Whisper API with `language="ja"`
- **How (Option B — Local):** whisper.cpp with the `large-v3` model for best Japanese accuracy
- **How (Option C — Quick):** Upload to a transcription service that supports Japanese medical audio
- **Output:** A text file containing the Japanese transcript
- **Time:** ~15 minutes
- **Quality check:** Skim the transcript. Are medical terms (medication names, conditions) captured correctly? Note any obvious errors.

### Step 3: Generate Report

- **What:** Process the transcript into a structured doctor briefing
- **How:** Use the prompt in [REPORT-TEMPLATE.md](REPORT-TEMPLATE.md) with Claude API, ChatGPT, or manual paste into a chat interface
- **Output:** A Japanese-language doctor briefing in four sections
- **Time:** ~30 minutes (including prompt iteration)

### Step 4: Self-Evaluate

Review the generated report against what you remember from the visit.

| Evaluation Criteria | Pass/Fail | Notes |
|---|---|---|
| Were all medications correctly identified? | | |
| Were all instructions captured? | | |
| Was the visit summary accurate? | | |
| Were follow-up items correctly identified? | | |
| Is Section 3 (次回診察に向けて) useful or noise? | | |
| Did it catch anything you forgot? | | |
| Would you be comfortable sharing this with your doctor? | | |
| Is the Japanese natural and appropriately clinical? | | |

**Time:** ~15 minutes

### Step 5: Doctor Feedback

- **What:** Share the report with your doctor at your next visit
- **How:** Print it, or email/LINE the secure link to the clinic beforehand
- **Ask the doctor:**
  - "前回の診察からAIで作成した報告書です。次回の診察の準備に役立つと思いますか？" ("This is an AI-generated report from our last consultation. Do you think it would be useful for preparing for the next visit?")
  - Was anything inaccurate?
  - Was anything missing?
  - Would you want to receive this before every visit?
- **Time:** 5 minutes during your next appointment

---

## Success Criteria

| Metric | Threshold | Kill Criteria |
|---|---|---|
| Transcription accuracy (medical terms) | >90% of key terms correct | <70% — transcription tech insufficient |
| Report factual accuracy | No critical errors (wrong medication, wrong instruction) | Any critical error — pipeline not trustworthy |
| Doctor usefulness rating | Doctor says "yes, I'd want this" | Doctor says "interesting but I wouldn't use it" |
| Patient willingness to pay | "I would pay ¥1,500/month for this" | "Nice but not worth paying for" |
| Section 3 (next visit prep) value | Identifies at least 1 genuinely useful follow-up insight | All observations are obvious or wrong |

---

## If Experiment Zero Succeeds

**Move 1 (Week 1-2):** Run 3-4 more recordings across different doctors and visit types. Confirm results are consistent.

**Move 2 (Week 2-4):** Have 5 other expats run the same experiment. Provide them the prompt and instructions. Collect their feedback.

**Move 3 (Week 4+):** Begin building the app. At this point you have real transcripts, real outputs, and real feedback to design against.

## If Experiment Zero Fails

Analyze *where* it failed:

- **Transcription failed** → Try different transcription model/service. Japanese medical audio is hard. May need specialized model.
- **Extraction was inaccurate** → Iterate on the prompt. Try different LLM. Add medical terminology context.
- **Report was accurate but doctor didn't care** → The value proposition needs rethinking. Doctor may not be the right audience for v1 output.
- **Everything worked but not worth paying for** → The problem may not be painful enough. Consider pivoting to a different wedge (e.g., multilingual patient summary first).

## Materials Needed

- A smartphone with any audio recording app
- A scheduled doctor's appointment
- Access to Whisper API or local Whisper installation
- Access to Claude or GPT-4 (API or chat interface)
- The prompt from [REPORT-TEMPLATE.md](REPORT-TEMPLATE.md)
- A printer or the ability to email/LINE the output to your clinic

**Total cost:** ~$1-2 in API fees. **Total time:** ~1 hour plus two doctor visits.
