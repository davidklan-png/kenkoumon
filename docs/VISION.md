# Kenkoumon — Vision & Roadmap

## The One-Sentence Vision

Every patient owns a complete, structured, portable record of their health — and uses it with any doctor or AI they choose.

---

## Stage 0: Experiment Zero (Now)
- **Duration:** 2-4 weeks
- **Goal:** Validate that a recorded Japanese consultation can produce a doctor-useful report
- **What exists:** A prompt, a manual pipeline, a printed report
- **Success criteria:** Doctor reads the report and says "this is useful"

## Stage 1: Expanded Validation (Month 2-3)
- Run across 3-5 different doctors and visit types
- Recruit 5 other expats to run the same experiment
- Success: >70% of doctors find it useful, >70% of test users would pay ¥1,500/month

## Stage 2: MVP App (Month 3-6)
- In-app recording, one-tap processing, patient review, secure link sharing
- Entity-centric data model (sessions populate entities)
- Target: English-speaking expats in Tokyo

## Stage 3: Longitudinal Profile (Month 6-12)
- Cross-visit insights, entity timeline, document ingestion (kenkou shindan, blood tests)
- Bilingual entity model fully populated

## Stage 4: Open Health Vault (Month 12-24)
- Structured data export (JSON + markdown + source documents)
- Tiered export with de-identification, PHI risk education
- Apple HealthKit import, API for patient-authorized AI access

## Stage 5: Platform (Year 2-5)
- Conversational AI query of health vault
- Clinic portal, insurance integration, international expansion
- FHIR export compatibility

## What We Are Not
- **Not a medical device.** We don't diagnose or recommend treatment.
- **Not a documentation tool.** We help the patient, not the doctor's paperwork.
- **Not a walled garden.** Patient data is portable and AI-agnostic.
- **Not a clinic management system.** Patients first. Clinic features secondary.
